# Emby Next Gen Console - 部署说明

## 环境要求

- Python 3.12+
- Node.js 18+
- PostgreSQL 14+
- Redis 7+
- Poetry（Python 依赖管理）

## 手动部署

### 1. 后端

```bash
cd backend

# 安装依赖
poetry install

# 配置环境
cp .env.example .env
# 编辑 .env，设置正确的数据库和 Redis 连接

# 初始化数据库
alembic upgrade head

# 运行
poetry run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 2. 前端

```bash
cd frontend

# 安装依赖
npm ci

# 构建生产版本
npm run build

# 预览构建结果
npm run preview
```

生产环境建议使用 Nginx 托管前端静态文件，并代理 API 请求到后端。

### Nginx 配置示例

```nginx
server {
 listen 80;
 server_name emby-console.example.com;

 root /var/www/emby-console/frontend/dist;
 index index.html;

 location / {
 try_files $uri $uri/ /index.html;
 }

 location /api {
 proxy_pass http://localhost:8000;
 proxy_set_header Host $host;
 proxy_set_header X-Real-IP $remote_addr;
 }
}
```

## 数据库迁移

```bash
cd backend
alembic current          # 查看当前版本
alembic upgrade head     # 升级到最新
alembic downgrade -1     # 回退一个版本
alembic revision --autogenerate -m "描述"  # 自动生成迁移
```

## 故障排查

### 后端日志
```bash
# 查看 uvicorn 日志
journalctl -u emby-console -f

# 或使用 docker logs
docker logs -f emby-console-backend
```

### 前端构建错误
- 确保 Node.js 版本 >= 18
- 清除 `node_modules` 和 `dist` 后重试

### 数据库连接失败
- 检查 `DATABASE_URL` 格式
- 确保 PostgreSQL 已启动并允许远程连接
- 验证用户权限和数据库是否存在

## 性能优化建议

1. **后端**
   - 使用 Redis 缓存热点数据（已集成）
   - 为频繁查询的字段加索引
   - 启用 Gunicorn + UvicornWorker 运行生产服务

2. **前端**
   - 启用路由懒加载（已实现）
   - 使用 CDN 托管静态资源
   - 开启 HTTP/2 和 Gzip
